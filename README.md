# Python DSA Practice Repository

This repository is a focused collection of Python solutions for Data Structures and Algorithms practice. It acts as a personal practice log, revision sheet, and problem-solving reference for common interview-style questions.

Rather than being a single project, this repo is a growing set of standalone scripts that capture how different problem types are approached, solved, and explained in Python.

## Why This Repo Exists

This repository is useful for:

- Practicing core DSA patterns through small, targeted problems.
- Building problem-solving consistency one question at a time.
- Revising important concepts before interviews, tests, or coding rounds.
- Comparing straightforward and improved approaches for the same problem.
- Keeping a record of learning progress in one place.

It is not just a folder of scripts. It is a compact archive of thinking patterns such as iteration, brute force, optimization, prefix sums, hashing, bit manipulation, sorting logic, and array transformations.

## Topics Covered

The current set of practice problems touches multiple DSA areas:

- Number series and pattern problems
- Prime numbers and Fibonacci logic
- String transformation
- Sorting and searching
- Array union and rotation
- Prefix sum and hashing
- Maximum subarray style problems
- Bit manipulation

| File | Problem Focus |

## What Makes This Repository Important

Small DSA repositories like this are valuable because they help turn theory into repeatable problem-solving skill.

- They improve coding speed through repetition.
- They make algorithmic patterns easier to recognize.
- They create a reliable revision base for future practice.
- They help track progress from basic logic to better optimizations.

Over time, a repository like this becomes a practical handbook for preparing for placements, technical interviews, coding assessments, and academic exercises.

## How To Run

Each file is a standalone Python script. Run any script from the repository root.

```bash
python 5_selection_sort.py
```

Some scripts take direct standard input, while others prompt for input in the console. Follow the expected input format described inside each file.

## Repository Style

This repo currently follows a simple practice-oriented structure:

- One problem per file
- Clear, direct Python implementations
- In some files, both optimal and non-optimal approaches are preserved for learning purposes
- Problem statements and examples are included in several scripts for quick revision
